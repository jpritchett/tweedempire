
  6 phases, each producing visible results you can check in the editor:

  1. Fluid sim + wind clouds — core 64x64 Stable Fluids simulation hooked into Simulation._step_sim() at 4 Hz. Clouds drift and thicken with simulated wind.
  2. Rain + wet terrain — GPUParticles3D rain following the camera, terrain darkens and gets shiny under rain via shader.
  3. Fog — Environment fog driven by weather data, sky horizon washes out.
  4. Snow/frost — Temperature layer, cold fronts swap rain for snow particles, terrain whitens on flat high ground.
  5. Storms + lightning — Lightning flashes light up clouds, ocean waves grow, more foam.
  6. Weather lifecycle + debug — Automated pattern spawning/expiry, F10 debug overlay.

  7 new files in src/weather/, 6 existing files modified (small, targeted changes to each shader + simulation.gd + world.gd + heightmap_terrain.gd).

│ Key Design Decisions                                                                                                                                                                                                                                                         │
│                                                                                                                                                                                                                                                                              │
│ - CPU fluid sim (not compute shader): 64x64 at 4 Hz = ~20K ops/tick, trivially fast in GDScript. Same algorithm ports to compute shader later if needed.                                                                                                                     │
│ - Texture-based shader interface: 64x64 sampler2D textures, bilinear interpolation gives smooth gradients for free.                                                                                                                                                          │
│ - Separate visual/sim RNG: Lightning and particles use non-deterministic RNG. Only fluid sim + pattern spawner use the seeded RNG. Multiplayer clients get identical weather data with slightly different cosmetic flourishes.                                               │
│ - Weather state in Simulation.state: Serializable for save/load and network sync.                                                                                                                                                                                            │
